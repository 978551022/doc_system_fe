# 个人文档系统

一个基于现代前端技术栈构建的个人文档系统，集成了智能聊天、API测试、文档管理等多种功能，提供了现代化的用户界面和流畅的用户体验。

## 系统界面

### 项目截图

系统采用了现代化的界面设计，主要包括左侧导航菜单和右侧主内容区域：

- **左侧导航菜单**：包含智能对话、文档列表、API接口测试、历史对话、设置、帮助和关于等功能模块
- **右侧主内容区域**：根据选择的功能模块显示相应的内容，如聊天界面、文档列表、API测试界面等

### 界面特点

- **响应式设计**：适配不同屏幕尺寸，提供良好的移动端体验
- **现代化UI**：采用Element Plus组件库，界面简洁美观
- **主题切换**：支持浅色主题、深色主题和跟随系统主题
- **流畅交互**：实时消息发送和接收，流畅的动画效果

## 技术栈

- **前端框架**：Vue 3（Composition API）
- **UI组件库**：Element Plus
- **HTTP客户端**：Axios
- **构建工具**：Vite
- **路由管理**：Vue Router 4
- **状态持久化**：localStorage

## 开发指南

### 代码规范

- 使用Vue 3 Composition API编写组件
- 组件命名采用PascalCase
- 变量和函数命名采用camelCase
- 提交代码时遵循Conventional Commits规范

### 代码质量规范

#### 代码检查

- **ESLint**：使用ESLint进行代码质量检查，确保代码符合语法规范和最佳实践
- **Prettier**：使用Prettier进行代码格式化，确保代码风格一致
- **TypeScript严格模式**：使用TypeScript严格模式，提高代码类型安全性
- **提交前检查**：提交代码前必须通过ESLint和Prettier检查

#### 代码质量要求

- **变量命名**：使用有意义的变量名，避免使用缩写和单字母变量
- **函数设计**：函数应该单一职责，避免过长的函数体
- **注释规范**：添加必要的注释，说明复杂逻辑和关键算法
- **错误处理**：添加适当的错误处理，避免程序崩溃
- **代码复用**：提取可复用的代码，避免重复代码

### 主题开发

项目使用CSS变量实现主题切换，主要主题变量定义在 `src/App.vue` 中，支持浅色主题和深色主题的切换。

### 组件结构

项目采用组件化开发，主要组件结构如下：

- **Header**：页面顶部导航栏，包含logo、主题切换等功能
- **Sidebar**：左侧导航菜单，包含各功能模块入口
- **MainContainer**：主容器组件，包含聊天区域和输入区域
- **ChatInput**：聊天输入组件，包含消息输入、文件上传等功能
- **ChatPage**：聊天页面组件，显示聊天记录

### 文档规范

#### 注释要求

- **公共组件**：必须提供使用说明和示例
- **复杂逻辑**：必须添加详细注释，说明实现思路和关键算法
- **接口函数**：必须描述功能、参数和返回值
- **类型定义**：必须说明用途和字段含义

#### 文档维护

- **README文件**：保持更新，包含项目介绍、功能特性、快速开始等信息
- **接口文档**：同步维护，确保与实际接口一致
- **组件文档**：提供组件使用示例和API说明
- **变更记录**：及时更新，记录版本变更内容

### 性能优化规范

#### 组件优化

- 使用 v-once 处理静态内容，避免不必要的重新渲染
- 使用 v-memo 优化列表渲染，减少不必要的组件更新
- 合理使用 computed 和 watch，避免不必要的计算和监听
- 避免不必要的组件重新渲染

#### 资源优化

- 图片资源使用合适格式和尺寸，避免过大的图片资源
- 组件按需加载，减少初始加载时间
- 路由懒加载，提高应用性能
- 第三方库按需引入，减少打包体积

### 安全规范

#### XSS 防护

- 尽量避免使用 v-html 指令，防止 XSS 攻击
- 对用户输入的内容进行转义处理，确保安全显示
- 配置内容安全策略，限制资源加载和脚本执行

#### 数据安全

- 敏感信息（如密码、API 密钥等）不存储在客户端
- API 请求必须进行权限验证，确保只有授权用户才能访问
- 所有 API 请求和页面访问都使用 HTTPS 协议，确保数据传输安全

### 开发流程规范

#### 功能开发流程

1. **创建功能分支**：从主分支创建功能分支，命名格式为 `feature/功能名称`
2. **更新类型定义**：如果需要，更新 TypeScript 类型定义
3. **实现业务逻辑**：实现功能的核心业务逻辑
4. **编写单元测试**：为功能编写单元测试，确保代码质量
5. **代码检查**：运行 ESLint 和 Prettier 检查代码质量
6. **提交代码**：提交代码，遵循 Conventional Commits 规范
7. **创建 Pull Request**：创建 Pull Request，等待代码审查
8. **合并代码**：代码审查通过后，合并到主分支

## 使用说明

### 智能聊天功能

1. 在聊天输入框中输入消息
2. 点击发送按钮或按回车键发送消息
3. 等待AI响应
4. 聊天记录会自动保存
5. 可以点击侧边栏的"新建聊天"按钮开始新的对话
6. 可以在设置中调整主题和字体大小

### API测试功能

1. 进入API测试页面
2. 输入OpenAPI文档地址
3. 获取接口列表
4. 选择要测试的API
5. 填写请求参数
6. 点击"发送请求"按钮
7. 在响应结果区域查看响应数据

### 文档管理功能

1. 进入文档管理页面
2. 可以查看、编辑和删除文档
3. 可以搜索和筛选文档
4. 支持文档分类管理

### 历史记录管理

1. 进入历史记录页面
2. 可以查看聊天历史和API测试历史
3. 支持历史记录搜索和筛选
4. 可以删除历史记录

```bash

```
